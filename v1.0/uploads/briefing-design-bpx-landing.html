Briefing de Design

Nova Landing Page BPX Group

Especificacoes de design, estrutura, animacoes GSAP, wireframe visual e guidelines tecnicas para a construcao de uma single-page profissional.

01 — Conceito Geral

Direcao Criativa

Uma single-page institucional com visual monocromatico preto e branco, tipografia oversized e animacoes de scroll que contam a historia do grupo BPX de forma cinematografica. A ausencia de cor cria um efeito de sofisticacao maxima — o contraste puro entre preto e branco transmite credibilidade, seriedade e confianca.


Inspiracao visual: a estetica monocromatica de sites como Linear, Vercel e Apple, onde a hierarquia visual e criada exclusivamente por peso tipografico, espacamento e tonalidades de cinza. O preto e branco funciona como um "terno bem cortado" — sem distracao, 100% profissional.

Tom Visual

Preto e branco puro. Monocromatico. Sem cores de acento. Hierarquia visual criada por peso tipografico, tonalidades de cinza (#000 a #FFF) e whitespace generoso.

Movimento

Animacoes com GSAP que guiam o olhar, nao que distraem. Scroll-driven storytelling. Cada secao se revela conforme o usuario desce a pagina. Smooth scrolling via ScrollSmoother.

Tipografia

Oversized headlines no hero e nos titulos de secao. Font moderna (Inter ou similar) para corpo. Pesos fortes (700-900) nos destaques. Contraste maximo texto/fundo.

Principio: Hierarquia por Luminosidade

Sem cores, a hierarquia visual depende inteiramente da escala de cinza. Headlines em #FFFFFF puro, corpo de texto em #AAAAAA, textos de apoio em #777777, labels e metadados em #333333. A diferenciacao entre as secoes (pilares, marcas) e feita por peso tipografico, tamanho e espacamento — nunca por cor.

04 — Stack GSAP — Plugins e Implementacao

Plugins Necessarios (todos gratuitos desde 2025)

ScrollSmoother Scroll suave nativo em toda a pagina. Cria a sensacao "buttery smooth" sem layout jank. Config: smooth: 1.5, normalizeScroll: true
ScrollTrigger Motor principal. Cada secao dispara animacoes ao entrar no viewport. Controla pin, scrub, snap e callbacks de todas as secoes.
SplitText Divide headlines em chars/words/lines para animacao de reveal letra por letra. Usado no hero e nos titulos de secao.
Flip Transicoes de estado entre layouts (ex: cards de marcas expandindo ao clicar). Animacao de layout sem calculos manuais.
Observer Detecta scroll, touch e pointer events para interacoes customizadas no hero (parallax sutil no mouse move).

Instalacao (CDN)

<!-- GSAP Core + Plugins (CDN) --> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/gsap.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/ScrollTrigger.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/ScrollSmoother.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/SplitText.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/Flip.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/Observer.min.js"></script>

Mapa de Animacoes por Secao

Hero SplitText no headline (chars, stagger: 0.03) + fromTo no subtitulo (opacity 0 → 1, y: 30 → 0) + Observer parallax sutil no background com mousemove. Easing: power3.out
Marcas ScrollTrigger + stagger nos cards (scale: 0.9 → 1, opacity: 0 → 1, stagger: 0.12). Trigger: top do card entra em 80% do viewport. Easing: back.out(1.4)
Pilares SplitText no titulo (words) + fromTo nos pillar cards (y: 60 → 0, opacity: 0 → 1, stagger: 0.15). Bordas superior dos cards animam largura 0% → 100%. Easing: power2.out
Numeros Counter animation com gsap.to + snap de 0 ate o valor final (ex: 0 → 4, 0 → 2029). Dispara quando a secao entra no viewport. Duration: 2s. Easing: power1.inOut
Contato Fade-in simples (opacity: 0 → 1, y: 40 → 0). Icones sociais com stagger: 0.1. Easing: power2.out
Global ScrollSmoother wrapping toda a pagina (smooth: 1.5). Navbar com blur/transparencia que aparece apos scroll > 100px via ScrollTrigger callback.

Exemplo de Codigo — Hero com SplitText

// Registrar plugins gsap.registerPlugin(ScrollTrigger, ScrollSmoother, SplitText); // ScrollSmoother global const smoother = ScrollSmoother.create({ wrapper: "#smooth-wrapper", content: "#smooth-content", smooth: 1.5, normalizeScroll: true, effects: true }); // Hero headline reveal const heroSplit = new SplitText(".hero-title", { type: "chars, words" }); gsap.from(heroSplit.chars, { opacity: 0, y: 40, rotateX: -40, stagger: 0.03, duration: 1, ease: "power3.out", delay: 0.3 });

Nota sobre o Board do Pinterest

O board pinterest.com/thiagofelipe1311/enterprise nao esta acessivel para leitura automatica (restricao do Pinterest). As referencias abaixo foram compiladas a partir de padroes tipicos de boards "enterprise" no Pinterest + pesquisa em Awwwards, Dribbble, Behance e landing.love.

Padroes Visuais Identificados em Boards "Enterprise"

Os boards corporativos/enterprise no Pinterest tipicamente salvam layouts com os seguintes padroes — que se alinham perfeitamente com o conceito proposto:

  • Dark Hero Full-Screen — Background escuro com headline branco oversized, subtitulo cinza e CTA com cor de acento
  • Card Grid com Hover — Cards de produtos/servicos em grid 3-4 colunas com efeito de elevacao no hover
  • Stats/Numeros Grandes — Secao de metricas com numeros oversized e labels discretos
  • Gradientes Sutis — Radial gradients escuros como fundo de secoes para criar profundidade
  • Sticky Navbar Minimalista — Navbar fixa com blur backdrop que aparece apos scroll
  • Icones com Glow — Icones com halo de cor nos pilares/features
  • Tipografia Variable — Fontes com peso variavel para criar hierarquia forte (300 a 900)
  • Footer Compacto — Footer minimalista com logo, links sociais e dados legais (CNPJ, licenca)
